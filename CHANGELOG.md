# 更新记录

本文件记录 DengDeng AI 的重要功能、兼容性修复与运维变更。

## 未发布

### 备份与版本更新

- 数据库备份新增可持久化的自动调度策略，支持启停、执行间隔、保留天数和最大份数；默认每 24 小时备份一次，保留 30 天且最多 30 份。
- 新增手动执行过期清理。自动清理严格限定为系统自动生成的快照，管理员手动备份永不自动删除。
- 版本检查新增提交级更新日志，显示标题、短提交号和提交时间；更新完成后保留本次上线日志。
- 数据库备份页补充下一次执行时间、来源标记和策略状态，版本更新页补充待更新内容与空状态。
- 移动端侧栏新增可触达的关闭按钮，避免遮罩点击落到下层菜单；深色模式主按钮改为高对比配色。

### 发布与部署

- 新增 GitHub Packages 容器发布流程，自动构建 Linux amd64/arm64 镜像并发布到 GHCR。
- 正式版本生成完整语义化标签和 `latest`，主分支生成 `edge`，所有构建保留 `sha-*` 标签。
- 容器包附带 SBOM、构建来源证明与 OCI 仓库元数据，支持 GitHub 供应链验证。
- Docker 构建启用 pnpm、Go 模块与编译缓存，并使用目标架构原生交叉编译，减少多架构发布时间。
- 固定 pnpm 版本，并仅允许构建前端所需的 esbuild 安装脚本，其余依赖脚本继续默认拒绝。
- Compose 默认可拉取已发布镜像，同时保留从当前源码构建的方式。

## v0.1.0 — 2026-07-18

### 并发控制与账号调度

- 新增用户、API 密钥、上游账号三级并发上限。`0` 表示不限制，旧数据升级后保持原有行为。
- 用户和密钥并发槽通过一次原子判断占用，避免只拿到部分资源后形成死锁；流式响应结束、客户端取消、上游失败和重试切换都会释放槽位。
- 上游账号达到并发上限后进入有界等待队列。队列已满或等待超时时返回 `429`、`Retry-After`，防止请求无限堆积。
- 管理端系统设置新增并发等待时间和队列深度；用户、密钥、账号的新建与编辑界面均可配置并发上限。
- Sub2API JSON 导入和浏览器 OAuth 创建账号时会保留并发配置。
- 调度器在同优先级账号间综合当前负载、延迟 EWMA、错误 EWMA 和最近选择时间，优先选择更空闲、更稳定的账号。

### 请求兼容与故障切换

- OpenAI Chat Completions、Responses、Codex OAuth 和 Anthropic 转 Responses 共用请求标准化逻辑。
- 自动补齐函数工具缺失的对象参数结构，并修复对象根节点下未声明类型的 `anyOf` / `oneOf` 分支，减少 Claude Code、Codex 等客户端的 `Invalid tool parameters`。
- Codex Responses Lite 请求强制关闭并行工具调用；无工具请求会移除无效的 `parallel_tool_calls`。
- 超长 Responses 输入项 ID 会稳定缩短并保留唯一性，避免上游长度校验失败。
- 扩展可切换账号的错误范围：支持在 `404`、`405`、`413`、部分能力不匹配的 `400/422` 以及既有鉴权、限流、服务端错误后尝试下一个账号。
- 普通请求参数错误仍直接返回客户端，避免无意义重试放大流量。

### 监控与排障

- 用量记录新增排队、调度、上游耗时和实际上游尝试次数。
- 响应新增 `Server-Timing` 与 `X-DengDeng-Upstream-Attempts`，并保留原有 `X-Request-ID`。
- 新增 `X-DengDeng-Trace-ID`，便于客户端与管理端日志关联。
- 运行监控新增“等待槽位”实时指标；管理员用量表和 CSV 导出显示详细阶段耗时及重试次数。
- 失败请求也写入统一用量日志，但不会产生费用，便于定位账号池耗尽和最终上游错误。

### 前端与安全

- 修复推广中心、推广管理和兑换码页面在接口返回 `null` 时读取 `length` 导致的白屏。
- 修复侧边栏推广中心图标的无效 SVG 路径。
- 将主题初始化逻辑移至同源静态脚本，兼容严格 CSP；`script-src` 无需放开 `unsafe-inline`。
- 监控页面卸载时会清理刷新定时器，避免离开页面后继续请求。

### 数据与兼容性

- 用户、API 密钥、上游账号新增并发字段；用量日志新增四项链路指标，由服务启动时的数据库迁移自动补齐。
- 空推广码、佣金和兑换码集合统一返回 JSON 数组，前端同时保留防御性归一化。
- 架构文档补充三级并发、排队、释放和单实例状态边界。当前实现适用于单网关进程，多副本部署前需将并发计数迁移到 Redis 等共享存储。

### 验证

- 后端：`go test ./...`
- 前端：`pnpm build`
- 覆盖并发占用与释放、队列满和超时、客户端取消、账号排队、`413` 故障切换、请求标准化、运行指标、严格 CSP 和空推广数据等回归场景。
